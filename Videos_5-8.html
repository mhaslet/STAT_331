Video 5) stringr: Working with Text Data

In this video, we’ll explore the stringr package, which provides a consistent and easy-to-use set of tools for working with character strings. We’ll load the stringr package and create a sample vector of text to work with for our examples.

  • allows us to clean data with regard to strings
# Load the stringr package
library(stringr)
# 
# Create a vector of sample sentences
sentences <- c(
  "The quick brown fox jumps over the lazy dog.",
  "R is a language for statistical computing.",
  "The year is 2025, and the sky is blue.",
  "Another sentence with the word dog."
)

# Access sentence 3
sentences[3]
[1] "The year is 2025, and the sky is blue."

str_detect(): Detect a Pattern

str_detect() returns a logical (TRUE/FALSE) vector indicating whether a pattern is found in each string.

  • allows us to determine if a particular pattern is within a string

  • done vector-wise

  • provides a vector of T/F according to each string in the vector

# Detect which sentences contain the word "dog"
str_detect(sentences, pattern = "dog")
[1]  TRUE FALSE FALSE  TRUE

str_replace_all(): Replace Patterns

str_replace_all() finds all matches of a pattern and replaces them with a new string.

  • allows us to replace something every time we see it
# Replace all instances of the word "dog" with "cat"
str_replace_all(sentences, pattern = "dog", replacement = "cat")
[1] "The quick brown fox jumps over the lazy cat."
[2] "R is a language for statistical computing."  
[3] "The year is 2025, and the sky is blue."      
[4] "Another sentence with the word cat."
  • There is also str_replace() that will only replace the first occurrence of the pattern in each string within the vector
str_replace("I love a dog named Duke, he is a good dog", pattern = "dog", replacement = "cat")
[1] "I love a cat named Duke, he is a good dog"

str_to_upper(): Change Case

Functions like str_to_upper(), str_to_lower(), and str_to_title() change the case of your strings.

  • str_to_title() capitalizes the first letter of each word
# Convert the first sentence to all uppercase
str_to_upper(sentences)
[1] "THE QUICK BROWN FOX JUMPS OVER THE LAZY DOG."
[2] "R IS A LANGUAGE FOR STATISTICAL COMPUTING."  
[3] "THE YEAR IS 2025, AND THE SKY IS BLUE."      
[4] "ANOTHER SENTENCE WITH THE WORD DOG."

str_subset(): Keep Strings with a Pattern

str_subset() is a useful shortcut that returns only the strings from your vector that contain a match for the pattern.

# Keep only the sentences that contain the word "dog"
str_subset(sentences, pattern = "dog")
[1] "The quick brown fox jumps over the lazy dog."
[2] "Another sentence with the word dog."

str_extract(): Extract a Pattern

str_extract() pulls out the first occurrence of a pattern. This is powerful when combined with regular expressions to find specific types of text, like numbers. The pattern \\d+ matches one or more digits.

  • \\d+ is looking for any number throughout the entire string

    • d stands for number, + is throughout the entire string

    • looking for \\d+ returns a corresponding vector of NA or the number it finds (as a character type)

      • if an individual string has multiple numbers, it returns the first occurrence

    • returns as a vector

  • There is also str_extract_all() that returns all numbers in a string as a list; if multiple strings are searched, returns multiple lists of numbers

    • returns a list
# Extract the number from our sentences vector
str_extract(sentences, pattern = "\\d+")
[1] NA     NA     "2025" NA

str_split(): Split a String

str_split() breaks a string into a list of smaller pieces based on a delimiter pattern.

  • provides a list for each of the strings in the vector and all the words within it
# Split the first sentence into a vector of individual words
# The pattern " " splits the string at each space
str_split(sentences, pattern = " ")
[[1]]
[1] "The"   "quick" "brown" "fox"   "jumps" "over"  "the"   "lazy"  "dog." 

[[2]]
[1] "R"           "is"          "a"           "language"    "for"        
[6] "statistical" "computing." 

[[3]]
[1] "The"   "year"  "is"    "2025," "and"   "the"   "sky"   "is"    "blue."

[[4]]
[1] "Another"  "sentence" "with"     "the"      "word"     "dog."

Video 6) forcats: Working with Factors

This video covers the forcats package, which is designed for working with factors. Factors are R’s way of handling categorical variables, and forcats provides tools to solve common problems, like changing the order of levels or re-grouping them.

# Load the forcats package
library(forcats)

# Create a sample factor (vector) representing survey responses
responses <- factor(c("Agree", "Disagree", "Agree", "Neutral", "Disagree", "Agree"))

# Outputs the vector and lists its levels in alphabetical order
responses
[1] Agree    Disagree Agree    Neutral  Disagree Agree   
Levels: Agree Disagree Neutral

fct_relevel(): Manually Reorder Levels

By default, factor levels are ordered alphabetically. fct_relevel() lets you move specific levels to the front, changing their order for plots and tables.

#The original order is alphabetical: "Agree", "Disagree", "Neutral"
levels(responses)
[1] "Agree"    "Disagree" "Neutral" 
# Move "Neutral" to be the second level, after "Agree"
fct_relevel(responses, "Neutral", after = 1)
[1] Agree    Disagree Agree    Neutral  Disagree Agree   
Levels: Agree Neutral Disagree

fct_infreq(): Reorder by Frequency

fct_infreq() is a useful shortcut to reorder factor levels so that the most common level appears first.

  • Allows us to sort levels by most frequent to least frequent in terms of the ordering of the factors

  • Important for ordering bar plots within a bar plot

# Reorder the levels based on how many times each one appears
fct_infreq(responses)
[1] Agree    Disagree Agree    Neutral  Disagree Agree   
Levels: Agree Disagree Neutral

fct_rev(): Reverse Level Order

fct_rev() simply reverses the current order of the levels.

# Reverse the order of the levels in our responses factor
fct_rev(responses)
[1] Agree    Disagree Agree    Neutral  Disagree Agree   
Levels: Neutral Disagree Agree

fct_recode(): Change Level Values

fct_recode() allows you to manually change the name of each level.

  • Specifying the name of each of the factors
# Change the names of the levels to something shorter
fct_recode(responses,
  "A" = "Agree",
  "D" = "Disagree",
  "N" = "Neutral"
)
[1] A D A N D A
Levels: A D N

fct_collapse(): Collapse Multiple Levels into One

fct_collapse() is useful for grouping several old levels into a single new level.

  • Allows you to collapse multiple levels into just one level

  • 2nd most useful

# Group "Agree" and "Disagree" into a new level called "Opinion"
fct_collapse(responses,
  Opinion = c("Agree", "Disagree"),
  No_Opinion = "Neutral"
)
[1] Opinion    Opinion    Opinion    No_Opinion Opinion    Opinion   
Levels: Opinion No_Opinion

fct_lump_n(): Lump Least/Most Common Levels

fct_lump_* functions are for collapsing the least/most common levels into a new “Other” level. fct_lump_n() keeps the n most frequent levels and lumps the rest.

  • Lump the n most/least common factors from a new complex_factor()

  • If the factor occurs n or more times, it’ll specify the level

    • If it doens’t appear n or more times, it’s specified as Other

  • Useful for a categorical variable that has many occurrences

# Create a more complex factor
complex_factor <- factor(c("A", "B", "A", "C", "A", "B", "D", "E"))

# Keep the 2 most frequent levels ("A" and "B"), and lump the rest into "Other"
fct_lump_n(complex_factor, n = 2)
[1] A     B     A     Other A     B     Other Other
Levels: A B Other

Step 3: Visualization with forcats and ggplot2

Now that our data is tidy, we can create an advanced plot. Let’s make a bar chart showing the average score for each major, ordered from highest to lowest.

fct_reorder(): Making Plots Better

fct_reorder() allows us to reorder the levels of a factor (like major) based on another variable (like the average score). This makes our plots much easier to interpret.

# First, calculate the average score per major
avg_scores_by_major <- full_student_data %>%
  group_by(major) %>%
  summarize(avg_score = mean(score))

head(avg_scores_by_major)
# A tibble: 2 × 2
  major      avg_score
  <chr>          <dbl>
1 Biology         90  
2 Statistics      84.5
# Create the bar plot, using fct_reorder to sort the y-axis
ggplot(avg_scores_by_major, aes(x = avg_score, y = fct_reorder(major, avg_score))) +
  geom_col(fill = "steelblue") +
  labs(
    title = "Average Quiz Scores by Major",
    x = "Average Score",
    y = "Major"
  ) +
    theme_bw()

  • Without factor_reorder(), the majors on the y-axis would be sorted from lowest score to highest score

    • If we use factor_reorder() by avg_score, it reorders the factors from highest score to lowest score

facet_wrap(): Creating Subplots

Now let’s go back to our full_student_data and create a more detailed visualization: a bar chart for each student, faceted by their major.

  • Other elements we can facet by:

    • grid, null, wrap
# Create a separate plot for each major
ggplot(full_student_data, aes(x = quiz, y = score, fill = name)) +
  geom_col(position = "dodge") +
  facet_wrap(~ major) +
  labs(
    title = "Quiz Scores by Student and Major",
    x = "Quiz",
    y = "Score"
  )

Techniques Showcase: Additional ggplot2 Enhancements with mtcars

Now we’ll explore some other useful ggplot2 functions using the mtcars dataset.

Setup: Create a base plot for the mtcars examples.

# We'll use this base plot for the following examples
base_plot <- ggplot(mtcars, aes(x = wt, y = mpg, color = factor(cyl))) +

base_plot

coord_flip(): Flipping Axes

coord_flip() swaps the x and y axes. This can be very useful for making horizontal boxplots or bar charts, which are often easier to read if you have long category labels.

# Create a boxplot and then flip the coordinates
ggplot(mtcars, aes(x = factor(cyl), y = hp)) +
  geom_boxplot() +
  coord_flip()

scale_color_manual(): Customizing Colors

This function gives you manual control over the colors used in your plot. You can specify the exact color for each level of your aesthetic mapping.

  • ggplot2 can change the scale of variables based on different components

  • scale_color_manual can be hex code colors as well

# Start with our base plot and add manual colors
base_plot +
  scale_color_manual(values = c("4" = "red", "6" = "273859", "8" = "005637"))

theme(): Customizing Plot Appearance

The theme() function is incredibly powerful and lets you customize almost any non-data element of your plot. Here, we’ll use it to move the legend to the bottom of the plot.

  • Can change the position of the legend, or remove it with “none”

  • theme() in console to see many different options for plot modifications

# Start with our base plot and move the legend
base_plot +
  theme(legend.position = "bottom")

geom_text(): Adding Text Labels

This geom adds text annotations directly onto the plot. It requires x, y, and label aesthetics. It’s often used with a filtered dataset to avoid cluttering the plot.

# Create a subset of data for labeling
label_data <- mtcars %>%
  rownames_to_column("car") %>%

After you’ve created the perfect plot, ggsave() allows you to save it to a file, like a PNG or PDF, with control over the size and resolution.

Video 8) lubridate: Working with Dates and Times

Working with dates and times can be tricky. The lubridate package makes it much simpler by providing functions to parse, manipulate, and extract information from date-time objects. We will create a sample data frame to demonstrate its most useful functions.

## Load the lubridate package
library(lubridate)
library(dplyr)

# Create a data frame with dates and times stored as character strings
event_log <- data.frame(
  event_name = c("Project Start", "Midterm Review", "Final Deadline"),
  start_date = c("2025-01-20", "2025-03-10", "2025-05-02"),
  start_time = c("01/20/2025 09:00:00", "03/10/2025 14:30:00", "05/02/2025 17:00:00")
)

  • Initial data is all string type, we don’t want that

  • We want it to recognize dates and times

1. ymd(): Parsing Dates

Lubridate’s parsing functions like ymd(), mdy(), dmy(), etc., automatically figure out and convert character strings into proper date objects. The function name corresponds to the order of the year, month, and day in your string.

## Parse the 'start_date' column, which is in Year-Month-Day format
parsed_df <- event_log %>%
  mutate(date_parsed = ymd(start_date))

parsed_df
      event_name start_date          start_time date_parsed
1  Project Start 2025-01-20 01/20/2025 09:00:00  2025-01-20
2 Midterm Review 2025-03-10 03/10/2025 14:30:00  2025-03-10
3 Final Deadline 2025-05-02 05/02/2025 17:00:00  2025-05-02
class(parsed_df$date_parsed)
[1] "Date"
  • Creates a new variable date_parsed of data type Date

2. mdy_hms(): Parsing Datetimes

Similarly, functions like mdy_hms() parse strings that contain both date and time components.

# Parse the 'start_time' column, which is in Month-Day-Year Hour:Minute:Second format
parsed_df_2 <- event_log %>%
  mutate(datetime_parsed = mdy_hms(start_time))

parsed_df_2
      event_name start_date          start_time     datetime_parsed
1  Project Start 2025-01-20 01/20/2025 09:00:00 2025-01-20 09:00:00
2 Midterm Review 2025-03-10 03/10/2025 14:30:00 2025-03-10 14:30:00
3 Final Deadline 2025-05-02 05/02/2025 17:00:00 2025-05-02 17:00:00
class(parsed_df_2$datetime_parsed)
[1] "POSIXct" "POSIXt"
  • Creates a new variable datetime_parsed of type POSIXct (date and time related data type)

3 & 4. wday() & month(): Extracting Components

Once you have a date-time object, you can easily pull out individual components like the day of the week, month, year, or hour.

# Create a parsed date column first
parsed_log <- event_log %>% 
  mutate(date_parsed = ymd(start_date))

# Extract the day of the week and the month
parsed_df_3 <- parsed_log %>%
  mutate(
    day_of_week = wday(date_parsed, label = TRUE),
    event_month = month(date_parsed, label = TRUE)
  ) %>%
  glimpse()

5. duration(): Measuring Time Spans

Duration objects represent an exact number of seconds. You can create them with functions like ddays(), dweeks(), or by subtracting two date-time objects.

  • Automatically creating Date variables
# Create two date objects
date1 <- ymd("2025-07-10")
date2 <- ymd("2025-08-25")

# Find the duration between the two dates
time_span <- interval(date1, date2)
print(as.duration(time_span))
[1] "3974400s (~6.57 weeks)"
# Create a duration of 3 days
three_days <- ddays(3)
print(three_days)
[1] "259200s (~3 days)"

6. now(): Getting the Current Time

The now() function returns the current system date and time.

# Get the current date and time
current_moment <- now()
print(current_moment)
[1] "2025-08-05 01:06:05 MDT"
